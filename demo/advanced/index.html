How to use

Click 🗣️ Speech tool in the top-right corner to open the toolbar. Press ▶️ play to have this page spoken. Use the ⏪ previous and ⏩ next buttons to jump between sentences, and ⏸️ pause to stop at any time.

Open ⚙️ Voice options to choose a voice, change the speed, or turn on scroll-to-follow so the page keeps the current sentence in view.

Edits

Insertions and deletions are read as plain text: price was 100 kr. 75 kr. after the discount.

Documentation

wc-speech is a small text-to-speech web component built on the Web Speech API. It reads text from another part of the page, follows the document's lang attributes, and highlights the current sentence and word when the browser supports custom highlights.

Preview / known limits

This is an early preview for developer feedback. The API and markup contract may change. See also README.md in the repository root.

  • Use one speech tool per page — speechSynthesis and CSS.highlights are shared globally.
  • Point target at readable content. wc-speech subtrees are skipped automatically; keep other controls outside the target when you can.
  • When speech synthesis is unsupported, controls are disabled with no message yet.
  • Pause cancels the current sentence; resume starts that sentence again from the beginning.
  • Word highlighting depends on browser, voice, and the Custom Highlight API.
  • No automated tests yet — manual testing only.

Who this is for

wc-speech is a simple in-page speech tool for sighted users who do not have text-to-speech software installed. It is not a replacement for assistive technology. People who rely on screen readers or built-in operating-system narration already have their own, more capable speech tools and will use those — this component is intentionally simple and stays out of their way.

Why it exists (Directive (EU) 2019/882)

The European Accessibility Act (Directive (EU) 2019/882) Article 13.2 encourages key information to be available through more than one sensory channel — for instance, text that can also be rendered orally. wc-speech provides an in-page spoken version of written content to help cover that "written and oral format" expectation for visitors who would not otherwise have a speech option.

That said, native browser and operating-system text-to-speech features — and dedicated assistive technology — generally do this more reliably than a per-site speech tool. Treat wc-speech as a pragmatic fill for a specific gap, not as a substitute for real assistive technology or a guarantee of legal compliance.

Expected markup

Inside <wc-speech>, the component expects a voice select, a rate input, a [popover] options panel, command buttons with commandfor and command, and optionally [data-speech-scroll] and [role="status"]. See the toolbar on this page, or the simple demo for a one-button setup.

Basic setup

Add the component, point target at the content that should be read, and wire buttons to the component with commandfor and command. 

<button type="button" commandfor="speech" command="--show-controls">
  Show speech tool
</button>

<wc-speech 
  id="speech" voice="speech-voice" rate="speech-rate" 
  target="#article" prefer-voice="Microsoft" hidden>

  <button type="button" commandfor="speech" command="--playpause">
    <span data-speech-face="play">Play</span>
    <span data-speech-face="pause" hidden>Pause</span>
  </button>

  <button type="button" commandfor="speech" 
    command="--previous-sentence">Previous</button>

  <button type="button" commandfor="speech" 
    command="--next-sentence">Next</button>

  <select id="speech-voice"></select>

  <input id="speech-rate" type="range" min="0" max="3.5" step="0.5" value="1">
</wc-speech>

<article id="article" lang="en">
  <h1>Readable content</h1>
  <p>This text can be spoken.</p>
</article>

Attributes

target
CSS selector for the content that should be read. Put controls outside the target when you can. wc-speech elements inside the target are skipped automatically.
voice
ID of a select element that should be populated with available voices.
rate
ID of a range input used for speech speed. The current demo uses 0 to 3.5.
prefer-voice
Optional text used to prefer matching voice names or voice URIs when the component auto-selects a voice, for example Microsoft for system voices on Windows. Users can still choose another voice manually.
scroll
Optional boolean. When present, the page follows along by scrolling the current sentence into view while reading. Honors prefers-reduced-motion (smooth scrolling is disabled when the user prefers reduced motion). This attribute is the default for the "Scroll to follow while reading" checkbox in the voice options, and users can toggle it there at runtime.
code-lang
Optional language used when reading pre and code blocks, since source code is normally English. Defaults to en and ignores the surrounding document language. An explicit lang attribute on the code element still wins.
label-play / label-pause
Optional accessible name for the play/pause button (used as aria-label and title unless the button has data-speech-manual-a11y). Defaults to Play and Pause. Visible icon and text live in your markup — see data-speech-face below.
data-speech-face="play" / data-speech-face="pause"
On child elements of a command="--playpause" button. The component toggles which face is shown and sets data-speech-action on the button. You supply icons, emoji, or text; wc-speech does not hardcode them. Icon-only buttons can use aria-hidden="true" on faces and rely on label-play / label-pause for the accessible name.
status-speaking / status-paused / status-finished
Optional text announced in the status region when reading starts or resumes, pauses, and finishes. Defaults to Speaking, Paused, and Finished.
hidden
Hides the control bar until it is opened with --show-controls.

Commands

--show-controls / --hide-controls
Show or hide the control bar. Hiding also stops speech and clears highlights.
--playpause
Start speech, pause the current sentence, or continue from the start of the current sentence. Pause uses cancel() rather than pause() for better cross-browser behaviour.
--previous-sentence / --next-sentence
Move backward or forward while speech is active.
--toggle-options
Show or hide the voice and speed options popover.

Language

Add a lang attribute to the page and to any content that changes language. The component resolves language from the element being spoken and its ancestors, so mixed-language content like <span lang="fr"> can use another voice when available.

Highlight colors

Override these CSS custom properties to change the word and sentence highlights: 

:root {
  --speech-word-bg: Mark;
  --speech-word-text: MarkText;
  --speech-sentence-bg: SelectedItem;
  --speech-sentence-text: SelectedItemText;
}

Worth knowing

  • The component needs browser support for speechSynthesis and SpeechSynthesisUtterance.
  • Word and sentence highlighting is progressive enhancement. Speech still works without the Custom Highlight API.
  • Word highlighting also depends on the selected voice. Some Chrome Google voices do not emit reliable word boundaries, so the component may fall back to sentence highlighting only. Use prefer-voice to prefer voices that behave better for your target browsers.
  • Native commandfor support is still new, so the component includes a click fallback for its own commands.
  • Image alt text is spoken as-is; empty alt is skipped. abbr title is read as the expanded form. time datetime can add a spoken date. pre and code are read in code-lang. wc-speech, form controls, script, style, and elements with aria-hidden="true" or hidden are skipped.
